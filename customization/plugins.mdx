## Overview

This configuration uses LazyVim as the base, with additional plugins customized for specific workflows including coding practice, note-taking, and competitive programming.

## Neovim Plugins

### Plugin Manager

This configuration uses [lazy.nvim](https://github.com/folke/lazy.nvim) through LazyVim for plugin management.

<Tip>
All plugins are lazy-loaded by default for optimal startup performance.
</Tip>

  <Tab title="Completion">
    #### Tab Completion Enhancement
    
    Custom configuration for nvim-cmp with Tab key completion behavior.
    
    ```lua
    {
      "hrsh7th/nvim-cmp",
      dependencies = {
        "hrsh7th/cmp-emoji",
      },
      opts = function(_, opts)
        local cmp = require("cmp")
        
        opts.mapping = vim.tbl_extend("force", opts.mapping, {
          ["<Tab>"] = cmp.mapping(function(fallback)
            if cmp.visible() then
              local entry = cmp.get_selected_entry()
              if not entry then
                cmp.select_next_item({ behavior = cmp.SelectBehavior.Select })
              else
                cmp.confirm()
              end
            else
              fallback()
            end
          end, { "i", "s" }),
          ["<CR>"] = cmp.mapping(function(fallback)
            cmp.abort()
            fallback()
          end, { "i", "s", "c" }),
        })
      end,
    }
    ```
    
    **Key Behaviors**:
    - `Tab`: Auto-selects first item if none selected, confirms if item is selected
    - `Enter`: Aborts completion and inserts newline (doesn't auto-complete)
    - Emoji completion support via cmp-emoji
    
    **Source**: `nvim/lua/plugins/tab-complete.lua`
    
    <Note>
    LuaSnip keys are disabled in this configuration to prevent conflicts.
    </Note>

## Tmux Plugins

Tmux plugins are managed by [TPM](https://github.com/tmux-plugins/tpm) (Tmux Plugin Manager).

### Installed Plugins

| Plugin | Description | Repository |
|--------|-------------|------------|
| **TPM** | Tmux Plugin Manager | `tmux-plugins/tpm` |
| **Catppuccin Tmux** | Catppuccin color scheme for Tmux | `dreamsofcode-io/catppuccin-tmux` |

### Configuration

```bash
set -g @plugin 'tmux-plugins/tpm'
set -g @plugin 'dreamsofcode-io/catppuccin-tmux'

# Initialize TPM (keep this at the bottom of tmux.conf)
run '~/.tmux/plugins/tpm/tpm'
```

**Source**: `tmux/tmux.conf:25-29`

<Note>
The official Catppuccin plugin (`catppuccin/tmux`) is commented out in favor of the Dreams of Code variant.
</Note>

### Installing Tmux Plugins

<Accordion title="Installation Steps">
  1. Install TPM if not already installed:
     ```bash
     git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
     ```
  
  2. Reload Tmux configuration:
     ```bash
     tmux source ~/.tmux.conf
     ```
  
  3. Install plugins by pressing `Prefix + I` (Ctrl+Space + I)
  
  4. Update plugins: `Prefix + U`
  
  5. Uninstall plugins: `Prefix + alt + u`
</Accordion>

### Adding New Plugins

To add a new Neovim plugin:

1. Create a new file in `nvim/lua/plugins/` (e.g., `my-plugin.lua`)
2. Add your plugin specification:

```lua
return {
  "username/plugin-name",
  lazy = true,  -- Optional: lazy load
  opts = {
    -- Plugin options
  },
}
```

3. Restart Neovim or run `:Lazy sync`

### Adding Tmux Plugins

To add a new Tmux plugin:

1. Add the plugin line to `tmux/tmux.conf`:

```bash
set -g @plugin 'username/plugin-name'
```

2. Reload Tmux configuration: `tmux source ~/.tmux.conf`
3. Install the plugin: Press `Prefix + I`
